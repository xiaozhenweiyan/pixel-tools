# 折线图重构与渐进式训练动画 Spec

## Why
当前折线图存在两个问题：(1) 鼠标 hover 时画布不断缩小（`setupCanvas` 在每次重绘时基于 `clientWidth` 重新设置尺寸导致递归缩小）；(2) 缺乏对回测过程的可视化，用户无法感知权重是如何从历史数据中学习出来的。本次改动将折线图重构为「全宽底部 + 自制滚动条平移 + 加减号缩放」的固定视口模式以彻底修复缩小 bug，并新增渐进式回测训练动画，让用户直观看到「用前 N 个数字预测第 N+1 个、逐步修正权重、最终预测」的全过程。

## What Changes
- **重构折线图布局**：将折线图从右侧主区移到页面底部，左右充满屏幕宽度，固定高度（如 420px）
- **修复 hover 缩小 bug**：画布显示尺寸由容器 CSS 决定（一次性设定），不再在每次重绘时读取 `clientWidth` 重设尺寸
- **自制滚动条平移**：在折线图区域实现像素风自制滚动条（水平 + 垂直），支持拖动平移查看数据的不同区域
- **加减号缩放**：折线图右下角放置一对 +/- 像素风按钮，点击缩放数据视口（X 轴方向缩放，影响可见数据点范围）
- **新增渐进式回测训练动画**：点击「预测」后，先执行训练阶段——用前 2 个数字预测第 3 个、前 3 个预测第 4 个……直到所有用户数字被预测完毕，期间动态更新权重
- **权重条形图排序动画**：训练过程中每次权重更新，方法条形按新权重平滑过渡到新位置（动画时长 ~400ms）
- **折线图渐进绘制动画**：训练过程中每个回测预测点以「绘制入场」动画出现，连接线渐进展开
- **训练进度指示器**：显示「训练中 step X / N」进度条
- **训练完成后**：执行最终预测（用全部数字预测下一个），显示融合预测结果
- **上传到 GitHub**：所有改动完成后推送到远程仓库 `xiaozhenweiyan/pixel-predictor`

## Impact
- Affected specs: `pixel-predictor-webapp`（初版 spec 仍有效，本次为增量增强）
- Affected code:
  - `/workspace/index.html` — 调整布局（折线图移至底部全宽，新增缩放按钮、训练进度 UI）
  - `/workspace/styles/pixel.css` — 新增底部图表布局、自制滚动条样式、训练进度条样式、动画相关样式
  - `/workspace/js/chart.js` — 重构 `drawLineChart` 为视口模式（pan/zoom），修复 hover bug，新增渐进绘制与排序动画支持
  - `/workspace/js/weights.js` — 新增 `computeIncrementalBacktest(methods, series)` 函数，返回每一步的回测结果与累积权重
  - `/workspace/js/app.js` — 新增训练动画编排逻辑（`runTrainingAnimation`），整合渐进回测、权重动画、折线图动画

## ADDED Requirements

### Requirement: 折线图全宽底部布局
系统 SHALL 将折线图面板移至页面底部，左右充满屏幕宽度（带 20px 外边距），固定高度 420px。

布局变化：
- 桌面端：上方为双栏（左主区输入+结果，右侧栏方法列表+权重条形图），下方为全宽折线图
- 移动端：纵向堆叠，折线图在最底部

#### Scenario: 桌面端布局
- **WHEN** 页面在 ≥900px 屏幕加载
- **THEN** 折线图面板位于页面底部，宽度充满屏幕（左右各 20px 边距），高度 420px
- **AND** 上方区域保持双栏布局（输入+结果 | 方法列表+权重条形图）

#### Scenario: hover 不再缩小
- **WHEN** 用户鼠标在折线图上移动
- **THEN** 画布显示尺寸保持不变，仅 tooltip 位置更新
- **AND** 不触发任何 `canvas.style.width/height` 或 `canvas.width/height` 的重新赋值

### Requirement: 自制像素风滚动条平移
系统 SHALL 在折线图内部实现自制滚动条（不使用浏览器原生滚动条），支持水平与垂直平移。

滚动条规格：
- 位置：水平滚动条在图表底部内侧，垂直滚动条在图表右侧内侧
- 样式：像素风（深色轨道 `#1a1a2e` + 白色 3px 边框 + 金色 `#ffd700` 滑块 + 硬阴影）
- 滑块大小：反映当前可见区域占总数据区域的比例
- 交互：鼠标拖动滑块平移视口；滑块外点击轨道可跳跃
- 数据超出视口时才显示滚动条；数据完全可见时隐藏

#### Scenario: 水平平移
- **WHEN** 数据点数量超过当前视口可见范围
- **THEN** 底部显示水平滚动条，拖动滑块可左右平移查看不同数据段

#### Scenario: 垂直平移
- **WHEN** 数据 Y 值范围超过当前视口可见范围（缩放后）
- **THEN** 右侧显示垂直滚动条，拖动滑块可上下平移

### Requirement: 加减号缩放
系统 SHALL 在折线图右下角提供一对 +/- 像素风按钮，点击缩放数据视口。

缩放行为：
- 位置：折线图区域右下角，两个按钮垂直排列或水平排列
- 样式：像素风按钮（白边 3px + 硬阴影 + 按压动画）
- `+` 放大：减少可见数据点数量（X 轴范围缩小），数据点变稀疏
- `-` 缩小：增加可见数据点数量（X 轴范围扩大），数据点变密集
- 缩放中心：以当前视口中心为基准
- 缩放范围：最小显示 3 个数据点，最大显示全部数据点
- 滚轮支持：鼠标滚轮在图表上滚动等价于 +/- 缩放（可选增强）

#### Scenario: 放大
- **WHEN** 用户点击 `+` 按钮
- **THEN** 可见数据点数量减少（如从 10 个减到 7 个），数据点在 X 轴上间距变大
- **AND** 滚动条滑块变长（反映可见比例增大）

#### Scenario: 缩小
- **WHEN** 用户点击 `-` 按钮
- **THEN** 可见数据点数量增加，数据点在 X 轴上间距变小
- **AND** 当可见范围等于全部数据时，隐藏水平滚动条

### Requirement: 渐进式回测训练动画
系统 SHALL 在用户点击「预测」后，先执行渐进式回测训练阶段，再执行最终预测。

训练流程：
1. 用户输入序列 `[a₁, a₂, ..., aₙ]`（n ≥ 2）
2. 训练阶段从 step 1 到 step (n-2)：
   - step k（k = 1, 2, ..., n-2）：用前 (k+1) 个数字 `[a₁, ..., a_{k+1}]` 预测第 (k+2) 个位置
   - 对每个方法：调用 `method.predict([a₁, ..., a_{k+1}])` 得到预测值
   - 实际值 = `a_{k+2}`，计算每个方法的绝对百分比误差 APE
   - 累积所有已完成 step 的 APE，计算累积 MAPE
   - 基于累积 MAPE 重新计算权重（`1/(MAPE+ε)` 归一化）
   - 触发权重条形图排序动画（条形平滑移动到新位置）
   - 触发折线图渐进绘制动画（在位置 k+2 绘制该 step 的各方法预测点 + 融合预测点）
3. 训练完成后，执行最终预测：用全部 n 个数字预测第 n+1 个位置
4. 显示最终融合预测结果

动画时长：
- 每个 step 约 600-900ms
- 权重排序过渡 400ms
- 折线图入场 300ms
- 总训练时长 ≈ (n-2) × 750ms

#### Scenario: 训练阶段执行
- **WHEN** 用户输入 `[1, 2, 3, 5, 8, 13]` 并点击「预测」
- **THEN** 系统先执行 4 个训练 step（n-2=4）：
  - step 1: 用 [1,2] 预测位置 3，对比实际值 3
  - step 2: 用 [1,2,3] 预测位置 4，对比实际值 5
  - step 3: 用 [1,2,3,5] 预测位置 5，对比实际值 8
  - step 4: 用 [1,2,3,5,8] 预测位置 6，对比实际值 13
- **AND** 每个 step 后权重条形图与折线图带动画更新
- **THEN** 训练完成后执行最终预测（用全部 6 个数字预测位置 7），显示融合预测值

#### Scenario: 短序列训练
- **WHEN** 用户输入 `[5, 10]`（n=2）
- **THEN** 训练阶段为空（n-2=0，无回测点），直接执行最终预测

#### Scenario: 训练进度指示
- **WHEN** 训练进行中
- **THEN** 页面显示「训练中 step X / N」进度条（像素风，金色填充）
- **AND** 「预测」按钮禁用，显示「训练中...」

### Requirement: 权重条形图排序动画
系统 SHALL 在训练过程中每次权重更新时，让方法条形平滑过渡到新位置。

动画规格：
- 当权重变化导致排序变化时，条形以 400ms 的 transition 平滑移动
- 条形位置使用 CSS `transform: translateY()` 或 Canvas 插值实现
- 条形长度（权重比例）也平滑过渡
- 训练完成后最终排序保持

#### Scenario: 排序变化
- **WHEN** step k 后方法 A 的权重超过方法 B
- **THEN** 方法 A 的条形平滑上移到 B 的位置，方法 B 平滑下移
- **AND** 动画时长 400ms，无闪烁

### Requirement: 折线图渐进绘制动画
系统 SHALL 在训练过程中每个 step 后，以入场动画绘制该 step 的预测点。

动画规格：
- 每个 step 的预测点以「缩放 + 淡入」入场（从 0 缩放到正常大小，透明度 0→1，300ms）
- 各方法预测点（半透明小方块）与融合预测点（红色方块）一起入场
- 连接前一个点到当前点的线段以「绘制」动画展开（dash offset 动画，300ms）
- 已绘制的点保持显示，不重绘

#### Scenario: 渐进显示
- **WHEN** 训练 step k 完成
- **THEN** 折线图在位置 k+2 处以入场动画显示该 step 的所有方法预测点与融合预测点
- **AND** 之前 step 的预测点保持显示

### Requirement: 训练后最终预测与图表
系统 SHALL 在训练完成后显示最终预测结果，并在折线图上以醒目方式标记最终预测点。

最终预测显示：
- 融合预测值显示在结果区（红色方块在第 n+1 位置）
- 折线图自动平移视口使最终预测点可见
- 训练进度指示器消失，「预测」按钮恢复可用

#### Scenario: 训练完成
- **WHEN** 训练阶段所有 step 完成
- **THEN** 系统执行最终预测（用全部 n 个数字），融合预测值显示在结果区
- **AND** 折线图平移使位置 n+1 可见，红色方块标记最终预测
- **AND** 「预测」按钮恢复可用，进度指示器消失

## MODIFIED Requirements

### Requirement: 折线图可视化（修订）
系统 SHALL 在原生 Canvas 上绘制折线图，采用「固定显示尺寸 + 数据视口」模式：

- 画布显示尺寸：由容器 CSS 决定（充满容器宽度，高度 420px），一次性设定，不在重绘时改变
- 数据视口：内部维护 `{ xMin, xMax, yMin, yMax }` 表示当前可见的数据范围
- 平移：通过自制滚动条或拖拽改变视口位置
- 缩放：通过 +/- 按钮改变视口范围（X 轴方向）
- Y 轴自动缩放：根据当前 X 视口内的可见数据点自动计算 Y 范围

包含元素（保留原版）：
- 输入序列：金色（#ffd700）实线 + 像素方块节点
- 融合预测点：红色（#ff4500）方块，醒目
- 各方法预测点：半透明小方块叠加
- 坐标轴：白色 3px 边框 + 等宽字体刻度
- 鼠标悬停 tooltip（基于当前视口坐标计算）

#### Scenario: 视口渲染
- **WHEN** 折线图绘制
- **THEN** 仅渲染当前视口 `{ xMin, xMax, yMin, yMax }` 范围内的数据点
- **AND** 视口外的数据点不绘制（性能优化）

#### Scenario: 平移后重绘
- **WHEN** 用户拖动滚动条平移视口
- **THEN** 折线图按新视口重绘，画布尺寸不变

## REMOVED Requirements

### Requirement: 旧版折线图响应式 resize 重绘
**Reason**: 旧版在窗口 resize 时调用 `renderLineChart` 重绘，但 `setupCanvas` 基于 `clientWidth` 重设尺寸导致 hover 时缩小。新版改为固定显示尺寸 + 视口模式，resize 仅触发一次重绘（不重设画布尺寸）。
**Migration**: 窗口 resize 时仅调用 `drawLineChart` 重绘当前视口，不调用 `setupCanvas` 重设尺寸（除非容器实际尺寸变化超过阈值）。
